# KCDL Issue Description Templates

KCDL 프로젝트의 이슈 생성 시 적용되는 Description 템플릿이다.
이 파일이 **런타임 SSOT**이며, 스크립트(`jira_dsl_lib.py`)가 파싱하여 섹션 순서·필수 섹션·Jira heading format을 결정한다.
MD를 직접 작성할 때도 이 파일의 MD 포맷을 따른다.

---

## Task (KCDL_Task_템플릿_v1)

- 템플릿 원본: [ISUTPL-2821](https://jira.daumkakao.com/browse/ISUTPL-2821)
- 항목 순서: **목표, 결과, 링크**
- 필수 항목: 목표, 결과
- Jira wiki 형식: `h2. 섹션명`
- required_sections: 목표, 결과

### Jira wiki markup

```
h2. 목표
 * // 구체적인 목표를 문장으로 작성 *- 필수*

h2. 결과
 * // 의사결정, 결과물 등 산출물에 대해 작성 *- 필수*

h2. 링크
 * // 연관된 외부 리소스 나열 (기획서, 현황시트, 아지트 업무요청, ...)
```

### MD 포맷

```markdown
## 목표
- 구체적인 목표를 문장으로 작성 - 필수

## 결과
- 의사결정, 결과물 등 산출물에 대해 작성 - 필수

## 링크
- 연관된 외부 리소스 나열 (기획서, 현황시트, 아지트 업무요청, ...)
```

### Ground rule

- 각 항목의 순서는 **목표, 결과, 나머지 순**으로 한다
- 각 섹션의 본문은 반드시 `- ` bullet list로 작성한다. 한 줄 평문을 사용하지 않는다
- 하위 항목은 `  - ` (2칸 들여쓰기 + `-`)로 작성한다

## Epic

- 항목 순서: **배경, 목표, 기대 효과, 개발범위, 요청 정보, 연동 정보, 관련 링크, 완료보고**
- 필수 항목: 배경, 목표, 기대 효과, 관련 링크
- Jira wiki 형식: `h2. 섹션명`
- 에픽은 프로젝트 특성에 따라 섹션 구성이 다를 수 있다
- 선택 섹션은 내용이 없으면 생략할 수 있다
- required_sections: 배경, 목표, 기대 효과, 관련 링크

### Jira wiki markup

```
h2. 배경
- // 필수 - 문제 상황, 필요성, 해결하려는 한계 요약

h2. 목표
- // 필수 - 달성하려는 핵심 목표

h2. 기대 효과
- // 필수 - 완료 시 기대되는 변화

h2. 개발범위
- // 선택 - 개발 범위, 대상 컴포넌트

h2. 요청 정보
- 요청일:
- 요청자 (기획):
- 요청자 (개발):
- 요청 조직:

h2. 연동 정보
- // 선택 - 데이터 소스, 외부 시스템 연동 정보

h2. 관련 링크
- // 필수 - 외부 리소스 나열

h2. 완료보고
- // 선택 - 완료 결과, 일정, 보고 정보
```

### MD 포맷

```markdown
## 배경
- 문제 상황, 필요성, 해결하려는 한계 요약 - 필수

## 목표
- 달성하려는 핵심 목표 - 필수

## 기대 효과
- 완료 시 기대되는 변화 - 필수

## 개발범위
- 개발 범위, 대상 컴포넌트 - 선택

## 요청 정보
- 요청일:
- 요청자:
- 요청 조직:

## 연동 정보
- 데이터 소스, 외부 시스템 연동 정보 - 선택

## 관련 링크
- 연관된 외부 리소스 나열 - 필수

## 완료보고
- TBD
```

### Ground rule

- 각 항목의 순서는 **배경, 목표, 기대 효과, 나머지 순**으로 한다
- 각 섹션의 본문은 반드시 `- ` bullet list로 작성한다. 한 줄 평문을 사용하지 않는다
- 하위 항목은 `  - ` (2칸 들여쓰기 + `-`)로 작성한다
- 선택 섹션은 내용이 없으면 생략할 수 있다

### 섹션명 별칭 (alias)

스크립트가 자동 정규화하는 별칭. MD에서 아래 변형을 사용해도 정규형으로 변환된다.

| 입력 | 정규형 |
|------|--------|
| 기대효과 | 기대 효과 |
| 작업 범위, 작업범위, 개발 범위 | 개발범위 |

## MD ↔ Jira 변환 규칙

MD ↔ Jira wiki 변환 시 아래 규칙을 따른다.

| 방향 | 변환 전 | 변환 후 |
|------|---------|---------|
| MD → Jira | `#` ~ `######` | `h1.` ~ `h6.` |
| Jira → MD | `h1.` ~ `h6.` | `#` ~ `######` |
| Jira → MD (레거시) | `☑️ 섹션명` | `## 섹션명` |
| Jira → MD (레거시) | `h3. ☑️ 섹션명` | `## 섹션명` |

### 공통

- MD 헤딩은 항상 `## ` (h2, 공백 1칸)으로 통일한다. `###`, `####` 등 하위 레벨을 사용하지 않는다
- `☑️`는 레거시 마커이며, 신규에는 사용하지 않는다
